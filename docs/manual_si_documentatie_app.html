1 Prezentare generala

Notebook-ul 03_main_app.ipynb este componenta centrala a proiectului de disertatie. El combina doua sisteme de AI intr-o singura interfata Gradio rulata din Google Colab:

🎙 Voice Conversion

Transforma vocea unui utilizator intr-o voce tinta (politician roman), pastrand continutul verbal dar schimband identitatea vocala. Foloseste modelul RVC v2 fine-tuned pe ~30 minute de audio per persoana.

🔍 Detectie Deepfake Audio

Analizeaza un fisier audio si estimeaza probabilitatea ca acesta sa fi fost generat de inteligenta artificiala. Produce o analiza temporala pe ferestre de 2 secunde + spectrograma Mel cu overlay vizual.

ℹ
Scop academic exclusiv. Sistemul a fost construit pentru demonstrarea capacitatilor si limitelor tehnologiilor de sinteza vocala si detectie deepfake. Nu este destinat inselaciunii sau uzului comercial.

HuBERT (facebook/hubert-base-ls960)

Extrage reprezentari de continut vocal (ce se spune, independent de cine spune). Se foloseste layer 9 din modelul transformer. In loc de fairseq (incompatibil cu Python 3.12), se foloseste un mock bazat pe libraria transformers.

RMVPE (rmvpe.pt)

Extrage pitch-ul (F0 — frecventa fundamentala) din audio cu precizie mare. Este metoda recomandata pentru RVC v2, superioara alternativelor harvest si crepe.

RVC v2 Generator

Retea neurala (VITS-based) antrenata sa genereze vocea tinta. Primeste feature-urile HuBERT + F0 si produce audio la 40kHz. Fiecare voce are un fisier .pth cu greutatile modelului.

FAISS Index (.index)

Index de cautare rapida care ajuta modelul sa reproduca mai fidel caracteristicile vocale din datele de antrenament. Controlat de parametrul Index Rate. Optional — daca fisierul lipseste, conversia merge fara el.

Fairseq Mock — de ce e necesar

RVC importa fairseq pentru a incarca modelul HuBERT. Insa fairseq nu este compatibil cu Python 3.12 (versiunea din Colab 2025+). Solutia: se injecteaza un modul Python fals in sys.modules['fairseq'] inainte ca RVC sa fie importat. Acest modul fals are aceeasi interfata API (load_model_ensemble_and_task) dar returneaza un obiect FakeHubertModel care in interior foloseste transformers.HubertModel.

3 Descrierea celulelor

Notebook-ul contine urmatoarele celule, in ordinea executiei:

ab000002 Mount Google Drive + verificare GPU
Monteaza Google Drive la /content/drive. Verifica si afiseaza GPU-ul disponibil (T4/A100/CPU). Trebuie sa ruleze primul — toate modelele sunt pe Drive.
ac000003 Instalare dependinte
Instaleaza toate pachetele necesare: gradio, librosa, soundfile, transformers, faiss-cpu, pyworld, praat-parselmouth, torchcrepe si altele. Instaleaza si ffmpeg prin apt-get. Dureaza 2-4 minute la prima rulare.
ad000004 Setup RVC — clone repo + download modele
Cloneaza repo-ul RVC la /content/RVC (daca nu exista deja). Descarca hubert_base.pt si rmvpe.pt din HuggingFace. Adauga RVC in sys.path.
7e4f1ff2 Fairseq Mock (HuBERT via transformers)
Instaleaza modulul fals fairseq in memorie. Descarca facebook/hubert-base-ls960 din HuggingFace. Clasa FakeHubertModel expune metoda extract_features(source, output_layer=9) compatibila cu API-ul fairseq. Trebuie sa ruleze inainte de orice import RVC.
af000006 VOICE_CONFIG + MODELS_DIR
Defineste MODELS_DIR, VOICE_CONFIG (dictionar cu informatii despre fiecare voce) si DEEPFAKE_MODEL_ID. Verifica si afiseaza ce modele exista pe Drive. Modifica aici calea DRIVE_BASE daca folderul tau Drive are alt nume.
e3b3fb07 Export modele din format checkpoint → format inferenta
Converteste fisierele .pth din formatul de training ({'model', 'optimizer', 'iteration'}) in formatul de inferenta asteptat de RVC ({'weight', 'config', 'f0', 'version', 'sr', 'info'}). Daca modelul e deja in format corect, nu face nimic. Ruleaza o singura data — modifica fisierele .pth pe Drive.
b0000007 Incarcare model detectie deepfake
Descarca si incarca MelodyMachine/Deepfake-audio-detection-V2 din HuggingFace. Detecteaza automat ce index din output corespunde clasei "fake". Dureaza 1-2 minute la prima rulare.
c0000008 Functia detect_deepfake()
Defineste functia de detectie. Incarca audio la 16kHz, segmenteaza in ferestre de 2s cu hop de 0.5s, ruleaza clasificatorul pe fiecare fereastra, calculeaza scorul global ponderat Hanning. Returneaza: (scor_global, fig_timeline, fig_spectrogram).
d0000009 Functia voice_convert()
Defineste functia centrala de conversie vocala. Seteaza variabilele de mediu, incarca modelul RVC la prima utilizare (sau la schimbarea vocii), apeleaza vc_single() si salveaza rezultatul ca fisier WAV temporar. Modelul e pastrat in cache (_rvc_instance) pentru a evita reincarcarile repetate.

Detalii tehnice importante:
  • cfg_rvc.is_half = False — dezactiveaza half-precision pentru a evita erori de tip float/half mismatch
  • sys.argv = [''] inainte de Config() — evita crash-ul argparse in Colab
  • vc_single() returneaza (status_str, (sample_rate, audio_int16)) — unpackingul corect e status_msg, audio_data = result
  • Conversia int16→float32: out.astype(np.float32) / 32768.0
f0000011 Interfata Gradio (UI)
Defineste layout-ul aplicatiei cu 3 tab-uri: Conversie Vocala, Detectie Deepfake, Despre. Conecteaza butoanele la functiile callback (vc_callback_radio, df_callback_new). Nu lanseaza serverul — doar defineste obiectul app.
a1000012 Lansare aplicatie
app.queue().launch(share=True, debug=False, quiet=False). .queue() este obligatoriu pentru a evita deconectarile pe inferenta lunga. share=True genereaza un URL public gradio.live valabil 72 ore.

5 Manual — Detectie Deepfake

Cum se foloseste

Deschide tab-ul "Detectie Deepfake" din interfata Gradio.
Incarca un fisier audio (WAV, MP3, FLAC etc.) — fie un clip real, fie unul generat de AI (de exemplu, rezultatul de la Voice Conversion).
Apasa "Analizeaza". Modelul proceseaza fereastra cu fereastra (2s cu hop 0.5s). Durata: 5-15 secunde in functie de lungimea audio.
Interpreteaza rezultatele: Apar 3 elemente — verdictul text, graficul temporal si spectrograma Mel.

Interpretarea verdictului

VerdictScorSemnificatie
PROBABIL AI-GENERAT> 75%Modelul e foarte confident ca audio-ul e sintetic
POSIBIL AI-GENERAT50% – 75%Semne de sinteza, dar nu concludent
POSIBIL REAL25% – 50%Mai probabil real, dar cu incertitudine
PROBABIL REAL< 25%Modelul e confident ca audio-ul este real

Interpretarea graficelor

Grafic temporal (Timeline)

Axa X = timp (secunde), Axa Y = probabilitate AI (0–100%). Linia rosie arata scorul per fereastra. Linia punctata = media globala. Un spike la un moment specific poate indica o portiune editata/inlocuita.

Spectrograma Mel + Overlay

Spectrograma standard Mel cu overlay colorat: rosu = segment suspect AI, verde = segment probabil real. Ajuta la identificarea vizuala a zonelor problematice.

⚠
Niciun detector de deepfake nu este 100% precis. Audio-ul convertit cu Voice Conversion poate sau nu sa fie detectat ca fake, in functie de calitatea modelului si caracteristicile sursei. Rezultatele trebuie interpretate ca indicative, nu definitive.

6 Parametri avansati — Voice Conversion

Accesibili din sectiunea "Setari avansate" de sub selectorul de voce.

🎵 Pitch Shift (semitonuri) — Range: -12 la +12, Default: 0

-12 (coboara)0 (neutru)+12 (urca)

Ce face: Transforma pitch-ul (inaltimea) vocii sursa inainte de conversie. Un semiton = diferenta minima perceptibila de pitch in muzica (o octava = 12 semitone).

Cand se foloseste: Cand vocea sursa si vocea tinta sunt de gen diferit. O voce de femeie (~200 Hz) trebuie coborata cu 8-12 semitone pentru a imita un barbat (~100-130 Hz). Invers pentru barbat → femeie.

Regula practica: Porneste de la valoarea din tabelul de la sectiunea 4, asculta, ajusteaza cu ±2 pana suna natural.

📁 Index Rate (Rata indicelui) — Range: 0.0 la 1.0, Default: 0.75

0.0 (fara index)0.5 (echilibru)1.0 (index maxim)

Ce este indexul: In timpul antrenamentului RVC, alaturi de modelul .pth se creeaza si un fisier .index — o baza de date (FAISS) cu caracteristicile vocale din datele de antrenament. La inferenta, modelul poate "consulta" aceasta baza pentru a reproduce mai fidel particularitatile vocii tinta.

Efectul valorii:

ValoareEfectCand se foloseste
0.0Indexul este ignorat complet — doar modelul .pthDaca indexul lipseste sau produce artefacte
0.5 – 0.7Echilibru — timbrulvocal e mai naturalVoce sursa foarte diferita de target
0.75 – 0.85Caracteristici vocale puternice ale tinteiRecomandat implicit — cele mai bune rezultate in general
0.9 – 1.0Index dominant — poate produce artefacte metalice/roboticeEvita daca audio-ul suna artificial
💡
Daca nu exista fisier .index in folderul de modele, parametrul este ignorat automat — conversia se face fara index si rezultatele sunt totusi bune.

🛡 Consonant Protection (Protectia consonantelor) — Range: 0.0 la 0.5, Default: 0.33

0.0 (fara protectie)0.25 (medie)0.5 (maxima)

Ce sunt consonantele nesonante: Sunete ca p, t, k, s, f, ș, h — produse fara vibratia corzilor vocale. Ele nu au pitch propriu-zis, deci conversia de pitch le poate distorsiona.

Ce face parametrul: Protejeaza segmentele fara pitch (consonante nesonante) de modificarea pitch-ului. Un segment protejat isi pastreaza caracteristicile originale in loc sa fie transformat.

Efectul valorii:

ValoareEfectCand se foloseste
0.0Zero protectie — toate segmentele sunt convertiteCand vrei conversie maxima, accepti distorsiuni pe consoane
0.33Echilibru — consoanele suna natural, identitate vocala bunaValoarea recomandata implicit
0.5Protectie maxima — consoanele suna clar, dar conversia e mai putin evidentaCand audio-ul sursa are multe consoane si suna distorsionat
✓
Regula simpla: Lasa la 0.33 implicit. Creste la 0.4-0.5 daca consoanele suna "metalice" sau distorsionate. Scade la 0.1-0.2 daca vrei ca vocea tinta sa fie mai dominanta.

Rezumat rapid — ce sa ajustezi

SimptomParametru de ajustatDirectie
Vocea suna prea inalta / pitigaiataPitch shiftScade (ex: -4)
Vocea suna prea joasa / groasaPitch shiftCreste (ex: +4)
Identitatea vocii tinta nu e claraIndex RateCreste (ex: 0.85)

7 Probleme frecvente si solutii

❌ "Modelul lipseste. Ruleaza 02_rvc_finetune.ipynb."

Fisierele voice_1.pth / voice_2.pth / voice_3.pth nu exista in /content/drive/MyDrive/disertatie/models/. Trebuie antrenate cu notebook-ul 02_rvc_finetune.ipynb (necesita GPU, 30-60 min per voce).

❌ "Conexiunea la server a fost pierduta"

Gradio a dat timeout pe o inferenta lunga. Solutia este deja aplicata in notebook: app.queue().launch(...). Daca se intampla totusi, ruleaza din nou ultima celula pentru un URL nou.

❌ Inregistrarea audio dispare sau nu se incarca

Problema cunoscuta cu componenta microfon Gradio in unele browsere. Solutii: (1) foloseste upload de fisier in loc de microfon; (2) incearca un alt browser (Chrome recomandat); (3) verifica permisiunile de microfon in browser.

❌ Eroare la rularea Runtime → Run All

Cel mai probabil Google Drive nu e montat inca. Asigura-te ca celula ab000002 a rulat si ti-a cerut autorizarea Google Drive. Re-ruleaza de la inceput dupa autorizare.

❌ Conversia dureaza 30+ secunde sau crapa

Rulezi pe CPU in loc de GPU. Verifica: Runtime → Change runtime type → T4 GPU. Dupa schimbare trebuie sa rulezi din nou toate celulele (Runtime → Run all).

❌ Vocile disponibile sunt 0 la celula af000006

Calea catre modelele de pe Drive este gresita. Verifica ca DRIVE_BASE = '/content/drive/MyDrive/disertatie' corespunde cu structura ta de foldere. Daca folderul tau se numeste diferit, modifica aceasta variabila in celula af000006.

❌ URL-ul Gradio a expirat

URL-urile gradio.live sunt valabile 72 de ore. Daca a expirat, ruleaza din nou ultima celula (a1000012) — nu trebuie sa rulezi tot notebook-ul, modelele sunt deja incarcate in memorie.
